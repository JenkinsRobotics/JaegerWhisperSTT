// what it is

The module IS the engine.

JaegerWhisperSTT is the stt-slot engine module for the Jaeger ecosystem. This package owns AudioSessionNode, the pywhispercpp-backed Whisper engine (two algorithmic modes), its own settings-catalog config slice, and its module.yaml manifest — the seam discover_modules() reads to bind the stt slot at boot.

Pins JaegerOS only — never JaegerAI — so a robot body can listen without the AI product installed at all. Two real consumers today: the JaegerAI product and JP01's non-AI console.

// capabilities

What ships in this module

Real, shipped capabilities — not aspirational ones.

Two-pass — "dual whisper"

A fast base.en model gates an accurate medium.en model: the fast pass makes wake-word matching responsive, the accurate pass commits the final transcript. A continuous mode is also registered.

engine

VAD + wake word

Voice-activity detection drives when a pass runs, not a fixed window. Wake-word gating (require_wake_word) is opt-in per instance.

audio

whisper.cpp backend

Via pywhispercpp — the same native engine that accelerates on Apple Silicon (Metal) without any Python-side GPU code.

core
// on deck — in active development

Decoupling the TTS dependency

The factory's ensure_tts_node() call (AEC reference-buffer sharing) is a real, previously-untestable coupling the split made visible — a decision is pending: graceful degradation without TTS installed, or accept the coupling by design.

// install

Two steps, then prove the contract.

1

Clone + install

Pins jaeger-os plus Whisper's own third-party libraries.

$ git clone https://github.com/JenkinsRobotics/JaegerWhisperSTT.git
$ cd JaegerWhisperSTT
$ pip install -e .
2

Prove the module contract

No microphone hardware or real Whisper model weights touched — the manifest parses, the node builds correctly-wired, the bus contract round-trips.

$ pytest jaeger_whisper_stt/nodes/whisper_stt/tests
# or:
$ python -m jaeger_whisper_stt.nodes.whisper_stt.tests.test_module_contract
// roadmap

From a plugin to its own repo.

0.8 M2b — Shipped

Graduated to an engine-module

The audio_session node and the Whisper engine consolidated into one package — no back-compat shims, every importer rewired to one place.

0.9.0 — Next

Standing on its own as a repo

Split out of the JROS monorepo with history preserved, renamed to jaeger_whisper_stt, packaged, and cross-install-proven against JaegerOS + JaegerAI + JaegerKokoroTTS in one venv — the out-of-tree loading milestone. The TTS coupling finding (above) surfaced here for the first time.

Later

1.0 — the ecosystem opens

An independently-versioned release once the stt slot contract survives a real body (JP01) and a real engine swap.
